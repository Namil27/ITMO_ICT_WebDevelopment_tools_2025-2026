# Задание 1  
# Различия между threading, multiprocessing и async в Python

## Постановка задачи

Необходимо реализовать три программы на Python, использующие:

- threading
- multiprocessing
- asyncio

Каждая программа должна вычислять сумму чисел от 1 до 100000000.  
Вычисления необходимо разделить на несколько параллельных задач и сравнить время выполнения.

---

## Использованные технологии

- threading
- multiprocessing
- asyncio
- time

---

## Реализация threading

В данной реализации использовались потоки Python из модуля threading.  
Задача разделялась между несколькими потоками, каждый из которых вычислял сумму своей части диапазона.

### Результат выполнения

- Result: 5000000050000000
- Time: 3.420816421508789

---

## Реализация multiprocessing

В данной реализации использовались процессы Python из модуля multiprocessing.  
Каждый процесс вычислял сумму отдельного диапазона чисел независимо от других процессов.

### Результат выполнения

- Result: 5000000050000000
- Time: 1.0983006954193115

---

## Реализация asyncio

В данной реализации использовалась асинхронность Python через asyncio.  
Вычисления были распределены между асинхронными задачами.

### Результат выполнения

- Result: 5000000050000000
- Time: 3.770878791809082

---

## Вывод по заданию 1

Наиболее быстрым способом выполнения вычислительной задачи оказался multiprocessing, так как процессы могут полноценно использовать несколько ядер процессора.

Threading показал худший результат из-за существования GIL (Global Interpreter Lock) в Python, который не позволяет потокам одновременно выполнять Python bytecode.

Asyncio также не дал прироста производительности, так как асинхронность предназначена в первую очередь для I/O-bound задач, а не для тяжёлых вычислений.

# Задание 2  
# Параллельный парсинг веб-страниц с сохранением в базу данных

## Постановка задачи

Необходимо реализовать три программы для параллельного парсинга веб-страниц с использованием:

- threading
- multiprocessing
- asyncio

Каждая программа должна:

1. Загружать HTML-страницы
2. Извлекать заголовок страницы
3. Сохранять данные в базу данных SQLite
4. Замерять время выполнения

---

## Использованные технологии

- requests
- aiohttp
- BeautifulSoup
- aiosqlite
- threading
- multiprocessing
- asyncio
- SQLite

---

## Структура базы данных

Использовалась база данных SQLite со следующей таблицей:

```sql
CREATE TABLE pages (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    url TEXT,
    title TEXT
);
```

---

## Реализация threading

В данной реализации для параллельной загрузки сайтов использовались потоки.

### Результат выполнения

- Threading time: 5.363905906677246

---

## Реализация multiprocessing

В данной реализации для параллельной обработки использовались процессы.

### Результат выполнения

- Multiprocessing time: 5.8632707595825195

---

## Реализация asyncio

В данной реализации использовались асинхронные HTTP-запросы через aiohttp и асинхронное подключение к базе данных через aiosqlite.

### Результат выполнения

- Async time: 4.999712944030762

---

## Вывод по заданию 2

Для задач, связанных с сетевыми запросами, наиболее эффективным оказался asyncio. Асинхронность позволяет выполнять ожидание ответов от серверов без блокировки программы.

Threading также показал хорошие результаты, так как сетевые запросы являются I/O-bound задачами.

Multiprocessing оказался менее эффективным, так как создание процессов требует дополнительных ресурсов операционной системы и не даёт преимуществ для сетевых операций.
